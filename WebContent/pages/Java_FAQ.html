Java 环境变量引用的问题

问题场景：

Windows 系统下，在配置好的 Java 环境变量后，不能使用 javac 命令。 检查 JAVA_HOME 及 PATH 变量，路径正常。PATH 中引用了 JAVA_HOME，换回全路径正常。

分析：

现象看来是 PATH 的问题，而 PATH 中引用了 JAVA_HOME，替换回全路径的 PATH 就没问题，说明对 JAVA_HOME 的引用出了问题。 最后发现唯一疑点是 JAVA_HOME 是设为用户变量，而 PATH 是系统变量，所以推测系统变量无法引用用户变量。

实验：

用户变量 系统变量
a = %b% b = B
c = C d = %c%

结果显示，a 可以正确引用 b，但 d 不能引用 c，即用户变量可以引用系统变量，而系统变量无法引用用户变量。

Windows 系统下的命名限制

我们知道，在 Windows 系统中，文件名不是任意无限制的，比如， \ / : * ? " < > | 等特殊符号是不能出现在文件名中的。

但是，有一些看上去比较普通的名称也是不能用来作文件名的，比如：

CON, PRN, AUX, CLOCK$, NUL, COM1~9, LPT1~9

它们是系统保留的设备名字，所以不能用作文件名。

不要尝试使用这些有限制的名称作为包名或类名，即使你是在其它操作系统上进行开发，而这些名称在其它操作系统上是允许的。 否则，代码移植到 Windows 系统进行开发或执行都可能存在问题。
